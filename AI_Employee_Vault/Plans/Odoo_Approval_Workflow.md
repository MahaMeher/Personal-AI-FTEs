# Odoo Approval Workflow Templates

## Overview

This document defines approval workflows for Odoo ERP operations in the Personal AI FTE system.

---

## Approval Thresholds

### Invoice Creation
| Amount | Action |
|--------|--------|
| < $100 | Auto-create draft, notify human |
| $100 - $500 | Create draft, require approval before posting |
| > $500 | Require approval before creation |

### Payment Processing
| Amount | Action |
|--------|--------|
| < $50 | Auto-process if recurring vendor |
| $50 - $500 | Require approval |
| > $500 | Require explicit human review + approval |

### Partner (Customer/Vendor) Creation
| Type | Action |
|------|--------|
| New customer | Auto-create, log for review |
| New vendor | Require approval (financial risk) |
| High-value partner (> $1000 transactions) | Flag for review |

## Workflow Examples

### Example 1: Invoice Creation Workflow

```
1. Email received: "Please send invoice for consulting work"
   ↓
2. Gmail Watcher creates Needs_Action file
   ↓
3. Qwen Code reads email, identifies invoice request
   ↓
4. Qwen Code uses @odoo to:
   - Check if customer exists (odoo_get_partners)
   - Create customer if needed (odoo_create_partner)
   - Create draft invoice (odoo_create_invoice)
   ↓
5. If amount > $500: Create approval request
   ↓
6. Human reviews and moves to /Approved/
   ↓
7. Qwen Code posts invoice (odoo_validate_invoice)
   ↓
8. Log action, move to /Done/
```

### Example 2: Vendor Bill Payment Workflow

```
1. Odoo Watcher detects unpaid vendor bill
   ↓
2. Creates Needs_Action/ODOO_VENDOR_BILL_*.md
   ↓
3. Qwen Code reads file, verifies:
   - Goods/services received
   - Amount matches PO (if applicable)
   - Payment terms
   ↓
4. Creates approval request (all payments require approval)
   ↓
5. Human reviews and moves to /Approved/
   ↓
6. Qwen Code processes payment via Odoo MCP
   ↓
7. Logs payment in Accounting/Current_Month.md
   ↓
8. Moves to /Done/
```

### Example 3: Overdue Invoice Follow-up

```
1. Odoo Watcher detects invoice overdue > 7 days
   ↓
2. Creates Needs_Action/ODOO_OVERDUE_*.md with draft email
   ↓
3. Qwen Code prepares payment reminder
   ↓
4. If first reminder: Auto-send (within policy)
   ↓
5. If second+ reminder: Create approval request
   ↓
6. Human approves escalation
   ↓
7. Qwen Code sends reminder via Email MCP
   ↓
8. Logs communication in Odoo via odoo_execute_kw
   ↓
9. Moves to /Done/
```

## Human-in-the-Loop (HITL) Rules

### Auto-Approved Actions (No Human Review)
- Creating draft invoices < $100
- Looking up Odoo data (read operations)
- Generating financial reports
- Creating new customers (low risk)

### Require Approval
- Posting invoices > $100
- Processing any payments > $50
- Creating new vendors
- Writing off bad debts
- Modifying posted entries

### Require Explicit Human Action
- Invoices > $500
- Payments > $500
- Refunds to customers
- Credit notes
- Journal entries affecting P&L > $1000

|------------|----------|-------------|-------------------|
| overdue_invoice | High | Draft reminder email | Sending 2nd+ reminder |
| unpaid_vendor_bill | Normal | Create payment draft | All payments |
| low_bank_balance | Critical | Alert human only | All actions |
| large_transaction | High | Flag for review | All transactions |
| invoice_approval_required | Normal | Create approval file | Posting invoice |
